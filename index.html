This is iJDBC

Make Oracle DBA's happy instrumenting JDBC connections

Instrument application code

A very important thing DBA's miss when monitoring an application is database calls, because we can trace application usage in an easy and quick manner. Without this instrumentation, it's not straight forward to know which module or action is actually being executed with an specific session, and check what it's doing. This behavior is common when database connections are created using a Connection Pool, because the database will be ignoring which end-user is actually using a session.

Learn more

What DBA's are used to seeing

From database perspective, we have following attributes to monitor an application: Client Identifier, Client Information, Module Name and Action Name. Those values can be read from the v$session view. There are other views, such as v$view which contain module and action attributes. We must be careful with v$sql view, because there could be shared sentences with module and action belonging to a different user, not the one who executed it afterwards.
